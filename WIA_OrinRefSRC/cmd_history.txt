Udev Rules: You can create a custom udev rule to set the correct permissions for the serial port device file when it's detected. This approach requires creating a udev rule file in /etc/udev/rules.d/. Here's an example:

Create a file, for example, 99-serial-ports.rules:

bash
Copy code
sudo nano /etc/udev/rules.d/99-serial-ports.rules
Add the following line to the file:

makefile
Copy code
KERNEL=="ttyTHS2", MODE="0666"
Save the file and restart udev:

css
Copy code
sudo udevadm control --reload-rules
Replug the device or reboot your system for the changes to take effect.

Try these solutions, and hopefully, one of them will resolve the permission issue, allowing your Python script to access the UART port without errors.
